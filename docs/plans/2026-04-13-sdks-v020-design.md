# Design: SDKs + v0.2.0 Release

**Date:** 2026-04-13
**Scope:** Rust, TypeScript, Java SDKs + README update + version bump
**Delivery:** Separate PRs per SDK, then README + version bump

---

## Context

README claims 6 SDKs (Rust, TS, Go, Java, Python, C) but `sdks/` directory doesn't exist. Primary consumers are Nucleus (Rust, library) and Orbit (Java, REST API). v0.2.0 covers 3 SDKs + 3 already-implemented features (Mailgun, webhooks, batch send).

## SDK 1: Rust (`sdks/rust/`)

**Crate:** `chorus-sdk`

Thin re-export wrapper over `chorus-core`. No new logic.

- Re-exports: `Chorus`, `SmsMessage`, `EmailMessage`, `SendResult`, `ChorusError`, `WaterfallRouter`, traits
- Prelude module: `use chorus_sdk::prelude::*`
- Dependencies: `chorus-core` only
- Doc examples in lib.rs

## SDK 2: TypeScript (`sdks/typescript/`)

**Package:** `@chorus/sdk`

REST client using native `fetch` (Node 18+ / Browser). Zero runtime deps.

```typescript
const chorus = new ChorusClient({ apiKey, baseUrl });

chorus.sms.send({ to, body, from? })
chorus.email.send({ to, subject, body, from? })
chorus.sms.sendBatch({ recipients, from? })
chorus.email.sendBatch({ recipients, from? })
chorus.otp.send({ to, appName? })
chorus.otp.verify({ to, code })
chorus.messages.get(id)
chorus.messages.list({ limit?, offset? })
chorus.webhooks.create({ url, events })
chorus.webhooks.list()
chorus.webhooks.delete(id)
```

- Auth: `Authorization: Bearer {apiKey}`
- Error: `ChorusError` extends `Error` with `status`, `code`
- Tests: vitest + msw

## SDK 3: Java (`sdks/java/`)

**Maven:** `com.chorus:chorus-sdk`

REST client using `java.net.http.HttpClient` (Java 11+). Jackson for JSON.

```java
ChorusClient chorus = ChorusClient.builder()
    .apiKey("ch_live_xxx")
    .baseUrl("https://api.chorus.dev")
    .build();

chorus.sms().send(new SendSmsRequest(to, body));
chorus.email().send(new SendEmailRequest(to, subject, body));
chorus.sms().sendBatch(new BatchSendRequest(recipients));
chorus.otp().send(new OtpSendRequest(to, appName));
chorus.messages().get(id);
chorus.webhooks().create(new CreateWebhookRequest(url, events));
```

- Builder pattern for client construction
- `ChorusException` for API errors
- Tests: JUnit 5 + WireMock

## SDK API Coverage

All 3 SDKs cover the same endpoints:

| Endpoint | Covered |
|----------|---------|
| SMS send / send-batch | Yes |
| Email send / send-batch | Yes |
| OTP send / verify | Yes |
| Messages get / list | Yes |
| Webhooks create / list / delete | Yes |
| Keys / Providers (admin) | No — v0.3.0 |

## Decisions

| Decision | Choice | Reason |
|----------|--------|--------|
| SDK count for v0.2.0 | 3 (Rust, TS, Java) | Cover consumers + largest community |
| Rust SDK approach | Re-export chorus-core | Nucleus uses library mode |
| TS HTTP client | Native fetch | Zero deps, works Node+Browser |
| Java HTTP client | java.net.http | Built-in Java 11+, zero HTTP deps |
| API scope | Core + webhooks | Admin endpoints via curl/dashboard |
| Go/Python/C | Deferred to v0.3.0 | YAGNI for current consumers |
